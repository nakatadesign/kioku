## Why kioku exists

If you use Claude Code seriously, you'll hit a wall.

The solution you found in yesterday's session is needed again today. The idea you had on your phone vanished by the time you opened your laptop. You're researching something you already looked up three months ago.

Claude Code is excellent in the moment. But **making knowledge persist across sessions** is something you have to build yourself.

kioku is a Git repository template that solves this.

It is not a tool. It is **a working directory for Claude Code to live in**.

---

## What kioku solves

| Problem | kioku's answer |
|---|---|
| Knowledge disappears when a session ends | Git persistence. Every session reads the same repository |
| Phone notes never make it into your workflow | Capture directly into Claude Code via Telegram |
| Past thinking stays buried | Proactive notifications surface connections: "This note links to something from 3 months ago" |
| Automation feels risky | 4-layer security design and full operation logs for post-hoc auditing |

## How knowledge flows

Every input lands in `inbox/` first.

```
в‘  Input (from any device)
       в”Ӯ
       в–ј
  inbox/YYYY-MM-DD-[type].md     вҶҗ auto-routed by trigger word
       в”Ӯ
       в–ј
  Claude Code session ends
  вҶ’ Stop hook: git commit (1 session = 1 commit)
  вҶ’ proactive-check: scan for connections to existing notes
       в”Ӯ
       в–ј
  /project:process (manual or scheduled)
  вҶ’ read inbox/, classify, tag
  вҶ’ organize and move to notes/[type]/
```

Prefix your message with a trigger word. Claude Code handles the rest.

| Message | Action |
|---|---|
| `idea: new feature concept` | appends to `inbox/YYYY-MM-DD-ideas.md` |
| `task: update docs by tomorrow` | appends to `inbox/YYYY-MM-DD-tasks.md` |
| `log: what I learned in this session` | appends to `inbox/YYYY-MM-DD-log.md` |
| `research: key points from this paper` | appends to `inbox/YYYY-MM-DD-research.md` |
| `save` | git commit + push |
| `status` | show today's inbox + git status |

## Proactive notifications вҖ” the core of kioku

Most PKM systems answer when you ask. kioku **reaches out to you**.

### Two trigger modes

**Scheduled (launchd)**
- Daily at 09:00 вҖ” Telegram morning brief: inbox items, related notes, active tasks
- Sunday at 10:00 вҖ” Knowledge quality scoring (HIGH/MEDIUM/LOW) + weekly report

**Contextual (Stop hook)**
- After every session, new inbox items are cross-referenced against existing `notes/`
- If a connection is found, Telegram fires: "This idea may connect to something from 2 months ago"

### Notifications that don't become noise

A system that over-notifies gets ignored. kioku has governance built in from the start.

```bash
MAX_DAILY=3          # max notifications per day
MIN_MATCHES=2        # minimum matching notes required to notify
QUIET_START=22       # silent hours start (22:00вҖ“07:00: no notifications)
QUIET_END=7
NOTIFY_COOLDOWN=86400  # same note won't trigger again within 24 hours
EXCLUDE_TAGS="#private #nopush #draft"  # excluded from notification
```

One variable change is all it takes to adjust behavior. Every notification's reasoning is recorded in `reports/proactive-YYYY-MM-DD.md` so you can tune false positives with full traceability.

## Security design

kioku uses `--dangerously-skip-permissions` for unattended Channels operation. Four defensive layers compensate for this:

```
[allowlist]           only your Telegram ID can issue commands
    вҶ“
[WorkingDirectory]    launchd is scoped to kioku/ вҖ” can't wander elsewhere
    вҶ“
[CLAUDE.md rules]     rm -rf, git push --force, and other destructive ops are prohibited
    вҶ“
[logs/]               all script stdout/stderr written to logs/
                      вҶ’ every action is auditable after the fact
```

Telegram is a third-party service. Notification messages contain only note titles and paths вҖ” never content, client names, amounts, or anything sensitive. Notes tagged `#private` are automatically excluded from all notifications.

## Optional: Semantic memory engine (v2)

kioku includes an optional local semantic search engine in `memory/`. When set up, it enables:

- **Meaning-based note discovery** вҖ” finds related notes even when keywords don't match, using a local embedding model ([Ruri v3-130m](https://huggingface.co/cl-nagoya/ruri-v3-130m))
- **Zero-cost proactive checks** вҖ” replaces the Claude API call in `proactive-check.sh` with local FTS5 + vector search
- **No external API dependency** вҖ” all search runs locally. Telegram notifications use the Bot API directly

This is entirely optional. Without it, kioku v1 works as-is with grep/glob search. To enable it:

```bash
bash scripts/memory-setup.sh
```

Requires Python 3.10+ with `enable_load_extension` support (e.g. Homebrew Python). See [`docs/memory-design-v2.md`](./docs/memory-design-v2.md) for architecture and setup details.
